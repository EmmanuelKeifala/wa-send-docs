## User registration

Use WhatsApp to deliver signup or email-verification codes immediately after account creation.

### Using templates (Recommended)

1. Create a reusable verification template with `POST /api/templates`.
2. Generate the code in your application.
3. Send the message with `POST /api/messages` using `templateId` and `templateArgs`.

Example template body:

```text
Welcome to wa-send. Your verification code is {{otp}}. It expires in {{expires}} minutes. Do not share this code.
```

Example send request:

```json
{
  "to": "+23276123456",
  "templateId": "template-verification-id",
  "templateArgs": {
    "otp": "483920",
    "expires": "5"
  }
}
```

### Direct messages (No templates)

Use direct messages if you are still iterating quickly and do not want to manage templates yet.

```json
{
  "to": "+23276123456",
  "message": "Welcome to wa-send. Your verification code is 483920. It expires in 5 minutes. Do not share this code."
}
```

## Password reset

Password reset flows should be short, explicit, and time-bound.

### Using templates (Recommended)

Example template body:

```text
Your password reset code is {{otp}}. It expires in {{expires}} minutes. If you did not request this, ignore this message.
```

Example send request:

```json
{
  "to": "+23276123456",
  "templateId": "template-password-reset-id",
  "templateArgs": {
    "otp": "271551",
    "expires": "10"
  }
}
```

### Direct messages (No templates)

```json
{
  "to": "+23276123456",
  "message": "Your password reset code is 271551. It expires in 10 minutes. If you did not request this, ignore this message."
}
```

## Two-factor authentication

2FA flows usually benefit from a single default template so every product surface sends the same wording.

### Recommended pattern

1. Create a dedicated 2FA template.
2. Mark it as default with `PUT /api/templates/{id}/set-default` if that fits your account workflow.
3. Pass only the current code and expiry at send time.

Example template body:

```text
Your security code is {{otp}}. It expires in {{expires}} minutes. Never share this code with anyone.
```

Example send request:

```json
{
  "to": "+23276123456",
  "templateId": "template-2fa-id",
  "templateArgs": {
    "otp": "604118",
    "expires": "5"
  }
}
```

## Implementation notes

### Error handling

| Status | Meaning | What to do |
| --- | --- | --- |
| `400` | Request body failed validation | Fix the payload before retrying |
| `401` | API key or bearer token is missing or invalid | Refresh credentials and retry |
| `404` | Template was not found for the authenticated account | Verify the template ID and ownership |
| `500` | Internal backend or queue issue | Retry only if your workflow treats the operation as transient |

### Best practices

- include an expiry window in every OTP-style message
- tell users not to share the code
- prefer templates for repeated product flows
- use `test` keys during development and `live` keys only in production
- treat `202 Accepted` as queued, not final delivery confirmation
